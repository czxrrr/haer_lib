# elinc_app
记得`push`前先`pull`，pull的时候注意对比一下有哪些行进行了修改，merge完成后再push。<br>
如果conflict就`手动merge`，冲突标志是：<br>
    <<<<<<< Updated upstream<br>
    ...1<br>
    =======<br>
    ...2<br>
    >>>>>>> Stashed changes<br>
    1部分是之前一个分支的代码，2部分是你的代码。<br>
    选择留下哪一部分之后再commit、push，冲突就解决。<br>
